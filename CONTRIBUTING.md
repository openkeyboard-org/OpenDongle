# Contributing to OpenDongle

## Getting started

1. Fork the repository and create a branch for your change.
2. Keep changes focused and scoped to a single issue.
3. Open a pull request with a clear description of what changed and why.

## Project structure

- `firmware/` for firmware/software changes.
- `hardware/` for hardware design changes.
- `tools/` for scripts and supporting tools.

## Licensing requirements

Contributions must follow the repository licensing model:

- Software contributions (firmware and tools) are under Apache License 2.0.
- Hardware contributions are under CERN-OHL-W-2.0.
